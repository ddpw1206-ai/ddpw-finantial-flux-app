# 1단계: 현재 구조 진단 보고서

## 📊 데이터 구조 분석

### 1. 월별 현황 `transactionData` 구조

**위치**: 전역 변수 `transactionData` (배열)

**데이터 소스**:
- `localStorage.getItem('transactionData')` 또는 빈 배열
- `saveData()` 함수로 `localStorage`에 저장

**데이터 필드**:
```javascript
{
  id: timestamp,
  date: 'YYYY-MM-DD',
  user: '담당자명',
  type: 'income' | 'expense',
  item: '항목명',
  category: '카테고리',
  amount: number,
  paymentMethod: 'credit' | 'debit' | 'transfer' | 'cash',
  paymentDetail: '결제수단 상세명',
  merchant: '사용처',
  detail: '세부내용',
  status: '상태',
  timestamp: number,
  // 추가 필드
  isCardPayment: boolean,  // 계좌 관리 카드 대금 표시
  isAutoGenerated: boolean // 자동 생성 여부
}
```

**사용 위치**:
- 월별 현황 모든 탭 렌더링
- 계좌 관리 입출금 내역 렌더링 (일부 필터링)
- 대시보드 통계 계산

---

### 2. 계좌 관리 데이터 구조

**위치**: 전역 변수 `accountData` (배열)

**데이터 소스**:
- `localStorage.getItem('accountData')` 또는 빈 배열
- `saveAccountData()` 함수로 `localStorage`에 저장

**데이터 필드**:
```javascript
{
  id: timestamp,
  name: '계좌명',
  type: 'bank' | 'card',
  balance: number,  // 초기 잔액
  owner: '소유자',
  // 카드인 경우
  cardType: 'credit' | 'debit',
  cardCompany: '카드사명',
  linkedAccount: '연결계좌',
  paymentDay: number
}
```

**계좌별 입출금 내역**:
- **현재 구조**: `transactionData`에서 필터링하여 사용
- **필터 조건**:
  - `isCardPayment === true` (카드 대금 청구) → 포함
  - `paymentMethod === 'transfer' || 'cash'` → 포함
  - `paymentMethod === 'credit' || 'debit'` → 제외 (단, `isCardPayment === true`는 예외)

## 🔄 렌더 함수별 데이터 사용 맵

### 1. `renderTable()` - 전체 소비 탭

**위치**: `index.html` 라인 4074

**데이터 소스**: `getCurrentMonthData()` → `transactionData` 필터링

**필터 조건**:
```javascript
monthData = getCurrentMonthData(); // transactionData에서 현재 월만 필터링
// 카드 대금 제외
monthData = monthData.filter(entry => {
  if (entry.isCardPayment === true) return false;
  if (entry.category === '카드대금' && entry.paymentMethod === 'transfer') return false;
  return true;
});
```

**표시 대상**:
- 모든 `type: 'income' | 'expense'` 항목
- 단, `isCardPayment: true` 제외
- 단, 카테고리 '카드대금' + paymentMethod 'transfer' 제외

**DOM 컨테이너**: `#total-content tbody` (직접 DOM 조작)

---

### 2. `renderCardTable(selectedCard)` - 카드별 내역 탭

**위치**: `index.html` 라인 4161

**데이터 소스**: `getCurrentMonthData()` → `transactionData` 필터링

**필터 조건**:
```javascript
monthData = getCurrentMonthData();
// 카드 대금 제외
cardData = monthData.filter(entry => {
  if (entry.paymentMethod !== 'credit' && entry.paymentMethod !== 'debit') return false;
  if (entry.isCardPayment === true) return false;
  if (entry.category === '카드대금' && entry.paymentMethod === 'transfer') return false;
  return true;
});
```

**표시 대상**:
- `paymentMethod === 'credit' || 'debit'` 항목만
- 단, `isCardPayment: true` 제외
- 선택된 카드로 추가 필터링 가능

**DOM 컨테이너**: `#card-table-container` (innerHTML로 `expense-table-container` 생성)

---

```javascript
monthData = getCurrentMonthData();

  return true;

```

**표시 대상**:
- `paymentMethod === 'transfer' || 'cash'` 항목만
- 단, `isCardPayment: true` 제외

**DOM 컨테이너**: `#bank-content` (innerHTML로 `expense-table-container` 생성)

### 4. `renderAccountTransactionTable(selectedAccount)` - 계좌 관리 입출금 내역

**위치**: `index.html` 라인 2953

**데이터 소스**: `getCurrentMonthData()` → `transactionData` 필터링

**필터 조건**:
```javascript
monthData = getCurrentMonthData();
monthData = monthData.filter(entry => {
  // 카드 대금 청구 항목은 포함
  if (entry.isCardPayment === true) return true;
  // 카드 사용 내역(credit/debit)은 제외
  if (entry.paymentMethod === 'credit' || entry.paymentMethod === 'debit') return false;
  // 계좌이체와 현금만 포함
  return entry.paymentMethod === 'transfer' || entry.paymentMethod === 'cash';
});
```

**표시 대상**:
- `isCardPayment === true` (카드 대금 청구) → 포함
- `paymentMethod === 'transfer' || 'cash'` → 포함
- `paymentMethod === 'credit' || 'debit'` → 제외 (단, `isCardPayment === true`는 예외)
- 선택된 계좌로 추가 필터링

**DOM 컨테이너**: `#account-transaction-table` (직접 DOM 조작, `expense-table-container` 사용 안 함)

## 📌 핵심 문제점 요약

1. **데이터 분리 부족**
   - 계좌 관리 카드 대금이 `transactionData`에 저장됨
   - 렌더링 단계에서 필터링으로 분리 시도 중 (불완전)

2. **`expense-table-container` 재사용 혼재**
   - `renderCardTable()`, `renderBankTable()`: `innerHTML`로 생성
   - `renderTable()`: HTML에 이미 정의된 컨테이너 사용
   - `renderAccountTransactionTable()`: 별도 DOM 구조 사용

3. **필터링 로직 중복**
   - 각 렌더 함수마다 유사한 필터링 로직 반복
   - `isCardPayment`, 카테고리 '카드대금' 체크가 여러 곳에 분산

4. **데이터 소스 구분 부재**
   - `sourceType` 같은 필드가 없어서 데이터 출처를 명확히 구분 불가
   - 레거시 데이터와 신규 데이터 구분 어려움
