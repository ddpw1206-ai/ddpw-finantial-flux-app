### ⚠️ 발견된 문제점

#### 1. **중복 정의 문제 (심각)**

**문제:**
- `APP_VERSION`이 `index.html`(1665줄)과 `config.js`에 중복 정의됨
- `transactionData`, `accountData`, `merchantHistory`, `cardData`가 `index.html`(1680-1683줄)과 `config.js`에 중복 정의됨
- `curYear`, `curMonth`가 `index.html`(1844-1845줄)과 `config.js`에 중복 정의됨

**영향:**
- 인라인 스크립트가 외부 스크립트보다 먼저 실행되어 `config.js`의 값이 덮어쓰여짐
- 데이터 불일치 및 예상치 못한 동작 발생 가능

**해결 방법:**
```javascript
// index.html의 인라인 스크립트에서 제거해야 할 부분:
// ❌ 제거: const APP_VERSION = '0.0.2';
// ❌ 제거: let transactionData = [];
// ❌ 제거: let accountData = [];
// ❌ 제거: let merchantHistory = [];
// ❌ 제거: let cardData = [];
// ❌ 제거: let curYear = ...;
// ❌ 제거: let curMonth = ...;
```

#### 2. **DOMContentLoaded 중복 실행 (심각)**

**문제:**
- `index.html`의 인라인 스크립트(1670줄)에 DOMContentLoaded 이벤트 리스너
- `app.js`에도 DOMContentLoaded 이벤트 리스너
- 두 개가 동시에 실행되어 충돌 가능

**영향:**
- 초기화 로직이 두 번 실행됨
- 이벤트 리스너 중복 등록
- 성능 저하 및 예상치 못한 동작

**해결 방법:**
- `index.html`의 DOMContentLoaded 내부 로직을 `app.js`로 이동하거나
- `index.html`의 DOMContentLoaded를 제거하고 외부 JS 파일만 사용

#### 3. **함수 정의 중복 (중간)**

**문제:**
- `saveData()`, `saveAccountData()`, `saveCardData()`가 `index.html`과 `data-manager.js`에 중복 정의
- `updateCategoryOptions()`, `updateItemOptions()` 등이 `index.html`과 다른 파일에 중복 정의 가능
- `expenseCategoryItems`, `incomeCategoryItems`가 `index.html`과 `config.js`에 중복 정의

**영향:**
- 함수 호출 시 예상치 못한 동작
- 코드 유지보수 어려움

**해결 방법:**
- 중복 함수를 외부 JS 파일로 통합
- `index.html`에서는 제거

#### 4. **인라인 스타일과 외부 CSS 충돌 가능성 (낮음)**

**문제:**
- `index.html`에 인라인 `<style>` 태그가 있음
- 외부 CSS 파일과 우선순위 충돌 가능

**영향:**
- 스타일이 예상과 다르게 적용될 수 있음

**해결 방법:**
- 인라인 스타일을 외부 CSS로 이동하거나
- 우선순위를 명확히 지정
