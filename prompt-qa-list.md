## 1. 탭 전환/페이지 렌더링 버그 자동 점검 ✅ 재수정 완료

탭(월별현황, 계좌 관리, 결제수단 관리, 저축관리 등)을 여러 번 전환해도  
이전 UI가 남아있거나 신규등록 버튼/FAB 팝업이 정상 작동하지 않는 등의 버그가 있습니다.

- index.html에서 각 탭 클릭 시 js 렌더 함수(renderDashboard, renderAccountsTab 등)가 제대로 동작하는지, main-content 초기화가 정상적으로 되는지 점검해 주세요.
- 각 js 및 이벤트 등록/스코프 연결, DOM 변화, global/window 연결을 자동 보정하고, 테스트 방법을 안내해 주세요.

**재수정 완료**: `app.js`의 중복 코드 제거, `renderAccountsTab` 호출 시 `accounts-container`가 없으면 `main-content`를 직접 사용하도록 수정, 모든 렌더 함수가 올바르게 호출되도록 보장

## 2. 수입/지출 구분별 드롭다운 옵션 오류 ✅ 수정 완료

입출금 내역 등록 화면에서 [수입]을 선택하면  
카테고리·항목 드롭다운에 지출용 옵션까지 같이 보이거나,  
올바른 분기(수입 시 수입항목만, 지출 시 지출항목만)가 동작하지 않아 저장 오류가 발생합니다.

- 수입/지출 선택에 따라 관련 옵션 데이터(배열/객체)와 렌더 함수가 완전히 분리되어 동작하도록 점검/수정해주세요.
- radio/select change 이벤트 리셋, renderCategoryOptions(), renderItemOptions() 등 함수별 코드 예시와 단위테스트 방법을 안내해 주세요.

**수정 완료**: 구분 라디오 버튼 변경 이벤트 리스너 추가, 카테고리/항목 드롭다운 완전 초기화 후 선택된 구분에 맞는 옵션만 추가하도록 수정

## 3. 데이터 실시간 동기화 및 파일 백업/복원

모든 데이터는 localStorage와 data/*.json 파일로 관리하고  
OneDrive 등 클라우드 폴더에 동기화합니다.

- File System Access API를 활용하여 JSON 파일로 자동 저장/백업/복원 기능을 구현
- 데이터 변경시 localStorage와 파일의 값이 즉시 반영되게 최적화하고,  
클라우드 폴더에서 동기화된 파일을 불러와 다른 PC에서도 완벽하게 활용할 수 있도록 구조를 안내해 주세요.

## 4. 거래내역 수정/삭제 버그 ✅ 수정 완료

거래 내역 수정 버튼을 눌렀을 때, '해당 내역을 찾을 수 없습니다.' 오류
- 배열/객체 연결, localStorage·JS 내 데이터 ID·키 구조 확인 및 버그 자동 점검
- 내역 저장/수정/삭제 시 데이터 무결성과 화면 동기화 테스트 방법 안내

**수정 완료**: `editTransaction` 함수에서 ID 타입 변환 로직 추가, `transactionData` 접근 전 유효성 검사 추가, 삭제 시에도 동일한 로직 적용

## 5. 계좌 관리, 입출금 내역 실시간 연동 ✅ 수정 완료

입출금 내역을 등록/수정/삭제하면
계좌 현황 및 월별 현황 탭에 자동으로 반영, 동기화되지 않는 문제가 있습니다.

- localStorage, 계좌, 입출금 내역 데이터 구조·렌더 함수 간 callback/observer 패턴으로 실시간 업데이트 최적화
- js 파일 간 함수 연결, 이벤트 트리거 구조, DOM 상태 변화 점검
- 테스트 방법 안내(동시 확인 체크리스트 포함)

**수정 완료**: 입출금 내역 등록/수정/삭제 시 `calculateAccountBalances()` 호출 및 현재 활성 탭 렌더 함수 자동 호출로 실시간 동기화 구현


## 6. FAB 버튼 - 결제수단 등록 팝업 동작 오류 점검 ✅ 수정 완료

FAB 버튼에서 [결제수단 등록]을 눌렀을 때  
결제수단 관리 페이지로 이동한 다음 뜨는 팝업이 '계좌 추가' 창(통장/초기잔액 입력)이 호출되고 있습니다.

- 실제로는 결제수단 관리에서 [+ 결제수단 추가] 버튼을 눌렀을 때와 동일한 "결제수단 추가(카드 등록)" 팝업 UI가 떠야 정상입니다.
- 현재 FAB 버튼 클릭 이벤트, 결제수단 관리 페이지 렌더 함수, 팝업 모달 연결 로직을 모두 점검해
- [계좌 추가] 팝업이 아니라 [결제수단 추가] 기능 화면이 동일하게 호출되는 구조로 수정해 주세요.
- 관련 js 함수(window/global), 팝업 초기화/랜더링 위치 점검과 이벤트 오류도 모두 보정 바랍니다.
테스트: FAB로 결제수단 등록→직후 팝업에서 카드 등록이 정상 동작, 내역 반영 여부까지 확인 바랍니다.

**수정 완료**: `openPaymentMethodModal` 함수가 `openCardManageModal`을 호출하도록 수정하여 결제수단 추가 모달이 올바르게 열리도록 구현

## 7. 계좌 관리 탭 - 입출금 후 잔액 반영 오류 점검 ✅ 수정 완료

계좌 관리 화면에서 수입이 발생하고 입출금 내역을 추가하면  
우측 '잔액' 부분이 0원으로 표기되는 문제가 있습니다.

- 잔액 칼럼에는 해당 계좌의 기존 잔액(또는 초기 잔액)에서 새로 추가된 입출금 내역을 모두 반영한 실제 잔액이 표출되어야 정상입니다.
- 입출금 등록/수정/삭제 등 데이터 변경시 계좌별 잔액(누적 계산)이 제대로 update되는지,
- 내역 테이블 연산, 계좌 데이터셋, localStorage 및 render 함수의 잔액 업데이트 callback 구조를 모두 자동 점검해 주세요.
- 문제 발견 시 잔액 계산/표기 로직을 보정하는 코드와 테스트 방법까지 안내 바랍니다.

테스트: 여러 입출금 내역을 등록/수정 시 잔액이 실시간으로 반영되고, 본 계좌의 잔고와 일치하는지 확인 바랍니다.

**수정 완료**: 입출금 내역 등록/수정/삭제 시 `calculateAccountBalances()` 호출 및 계좌 목록 갱신으로 잔액이 실시간 반영되도록 구현

## 11. 계좌 관리 탭 - 잔액 표기/업데이트 오류 ✅ 수정 완료

계좌 관리에서 입출금 내역을 추가(수입/지출 등)했는데
- 표 상단의 통장 잔고와
- 아래 상세 입출금 내역 테이블 우측(잔액) 칼럼
모두 잔액이 0원으로 계속 표기되는 문제가 있습니다.

기대 동작:
- 각 입출금 내역이 추가/수정/삭제될 때마다 해당 계좌별 현재 잔고가 실시간으로 반영되어야 합니다.
- 상세 테이블에도 시점별로 잔액이 누적 반영(직전 잔액→신규 입출금→현재 잔액)되어야 합니다.
- 전체 계좌 합계도 실시간 반영

점검 요청:
- index.html과 연결된 js 파일(특히 accounts.js, dashboard.js 등)에서
- 잔액 누적 계산 함수, 내역 렌더링, localStorage 및 데이터 업데이트, UI 표현, 수식 충돌, 이벤트/함수 이름 중복 등 모두 점검해 주세요.
- 문제 발견 시 코드/로직을 자동으로 개선하고, 수식/콜백/표기/테스트 방법까지 안내 바랍니다.

테스트: 여러 입출금 입력 후 상단과 내역 테이블 잔액이 맞게 실시간 반영되는지 확인하세요.

**수정 완료**: 
- `calculateAccountBalances()` 함수가 `paymentMethod === 'transfer' || paymentMethod === 'cash'`인 거래만 처리하도록 수정
- `renderAccountTransactionTable`에서 테이블 잔액 계산 시 `account.currentBalance`에서 현재 월 거래를 역순으로 빼서 초기값 계산하도록 개선
- 입출금 내역 추가/수정/삭제 시 `calculateAccountBalances()` 호출 및 계좌 카드/테이블 갱신으로 실시간 반영 보장
