### 3-3. 추가/수정/삭제 기능

A. 단건 추가 (모달)

입력 순서
1. 사용일자 [캘린더]
2. 사용자 [파우][둠둠][공용]
3. 구분 [수입][지출] → 자동 카테고리 필터링
4. 대분류 [드롭다운] → 자동 소분류 필터링
5. 소분류 [드롭다운]
6. 사용처 [텍스트] (선택)
7. 세부내용 [텍스트] (선택)
8. 금액 [숫자]
9. 결제 방법 [드롭다운] → 자동 세부수단 필터링
10. 세부 결제수단 [드롭다운]
11. 고정 지출 여부 [체크박스]
    → 체크하면 "시작월" "종료월" 입력 필드 표시
12. [저장] [취소]

검증 로직
- 필수 필드: date, user, type, mainCategory, subCategory, amount, paymentMethod, paymentDetail
- 금액 유효성: 양수만 허용 (0 제외)
- 중복 검사:
  date, user, mainCategory, subCategory, paymentDetail, amount, timestamp(시간) 모두 동일 시
  경고: "동일한 거래가 최근에 입력되었습니다. 정말 추가하시겠습니까?"

고정 지출 추가
고정 지출 체크 시:
└─ 시작월: [2025년 1월]
└─ 종료월: [2025년 12월] (또는 "계속")

저장 시:
1. FIXED_TRANSACTION에 저장
2. 시작월부터 종료월까지 각 월별로 TRANSACTION 자동 생성
3. 각 TRANSACTION에 fixedTransactionId 매핑
4. 월별 현황 재계산

B. 일괄 추가 (XLSX 파일)

지원 은행/카드사
- KB국민은행
- KB카드
- 삼성카드

프로세스
1. "일괄 입력" 버튼 클릭
2. XLSX 파일 선택
3. 컬럼 자동 감지 (추후: 수동 매핑)
4. 라인별 검증
5. 실패 라인 목록 표시
6. 실패 라인 상세 보기
7. 실패 사유 표시
8. 수동 수정 또는 스킵 선택
9. "사용처 자동 카테고리 제안"
10. 사용자 확인 후 저장
11. 일괄 저장

C. 수정

수정 방식: 인라인 수정 (행에서 직접 수정)
테이블 행 클릭 → 해당 필드들이 수정 가능하게 변함
→ 수정 내용 입력
→ [저장] 버튼 클릭
→ 검증 + 중복 검사 (자신 제외)
→ 저장 완료 → updatedAt, updatedBy 자동 기록
→ 월별 현황 재계산

제약사항
- 모든 사용자가 수정 가능
- 수정 이력 기록 (updatedAt, updatedBy)
- 고정 거래로 등록된 거래를 수정 → 해당 월만 변경 (고정 목록 미영향)

D. 삭제

개별 삭제
테이블의 [삭제] 버튼 클릭
→ 확인 다이얼로그: "정말 삭제하시겠습니까?"
→ 삭제 확인 → 거래 삭제
→ 월별 현황 재계산
→ 고정 거래로 등록된 경우: 해당 월만 삭제 (고정 목록 유지)

일괄 삭제

→ 전체 선택: 하단 "전체 선택" 체크박스
→ 개별 선택: 각 행의 체크박스
→ [일괄 삭제] 버튼 클릭
→ 확인 다이얼로그: "선택한 [n]개의 거래를 삭제하시겠습니까?"
→ 삭제 확인 → 모두 삭제
→ 월별 현황 재계산

  - 실제 통장 확인
  - 오류 수정
  - 기타

  - 이전달 거래는 변동 없음

- 거래 삭제: 즉시 순자산 변경
- 고정 거래 추가: 해당 월부터 순자산 변경

계산 로직:
월별순자산 = 전월말잔액(또는 설정잔액) + 해당월수입 - 해당월지출
