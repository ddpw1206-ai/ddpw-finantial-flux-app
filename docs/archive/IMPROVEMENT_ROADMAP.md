# 점진적 개선 로드맵

## 🎯 우선순위별 개선 계획

### Phase 1: 핵심 문제 해결 (최우선)
**목표**: 계좌 관리 카드 대금이 월별 현황에 표시되지 않도록 완전 분리

**작업**:
1. `accountCardPayments` 배열 생성 및 초기화
2. `js/modal.js` 카드 대금 저장 경로 변경
3. `renderAccountTransactionTable()` 수정하여 `accountCardPayments` 조회
4. 저장/불러오기 함수 추가

**예상 시간**: 1-2시간
**테스트**: 계좌 관리 카드 대금 등록 후 월별 현황 확인

---

### Phase 2: 필터링 로직 통합
**목표**: 중복된 필터링 로직을 공통 함수로 통합

**작업**:
1. 공통 필터 함수 생성 (`filterTransactionsBySource`)
2. `renderTable()`, `renderCardTable()`, `renderBankTable()`에서 공통 함수 사용
3. 필터 조건 명확화

**예상 시간**: 1시간
**테스트**: 각 탭에서 올바른 데이터만 표시되는지 확인

---

### Phase 3: sourceType 필드 추가 (하위 호환)
**목표**: 데이터 출처를 명확히 구분할 수 있도록 필드 추가

**작업**:
1. 신규 등록 시 `sourceType` 필드 추가
2. 자동 입력 시 `sourceType: 'AUTO_CARD'` 설정
3. 레거시 데이터 마이그레이션 (기본값 'MONTHLY')
4. 필터링 로직에 `sourceType` 반영

**예상 시간**: 1-2시간
**테스트**: 기존 데이터 유지, 신규 데이터에 필드 추가 확인

---

### Phase 4: 중복 체크 로직 개선
**목표**: 중복 체크를 통합 함수로 개선하고 UX 개선

**작업**:
1. `checkDuplicateTransaction()` 함수 생성
2. 월별 현황 수동/자동 입력에 적용
3. 사용자 확인 메시지 개선

**예상 시간**: 1시간
**테스트**: 중복 데이터 입력 시 경고 확인

---

### Phase 5: 레거시 필드 정리 (선택적)
**목표**: `isCardPayment`, `isAutoGenerated` 필드 제거

**작업**:
1. 모든 코드에서 `sourceType` 사용으로 전환
2. 레거시 필드 참조 제거
3. 데이터 마이그레이션 스크립트

**예상 시간**: 1시간
**테스트**: 기존 데이터 정상 동작 확인

---
