# 2단계: 데이터 모델 분리 설계

## 🎯 분리 원칙

### 원칙 1: 계좌 관리 카드 대금
- **저장 위치**: `transactionData`에 **절대 저장하지 않음**
- **대안**: 별도 배열 `accountCardPayments` 또는 `accountData[x].cardPayments` 배열 사용

### 원칙 2: 월별 현황 자동 입력 (카드 명세서 파싱)
- **저장 위치**: `transactionData`에만 저장
- **계좌 잔액 영향**: 없음 (계좌 잔액 계산에서 제외)
- **구분 필드**: `sourceType: 'AUTO_CARD'`

### 원칙 3: 월별 현황 수동 입력
- **저장 위치**: `transactionData`에 저장
- **계좌 잔액 영향**: 
  - `paymentMethod === 'transfer' || 'debit'` → 계좌 잔액 반영 ✅
  - `paymentMethod === 'credit' || 'cash'` → 계좌 잔액 반영 안 함 ❌
- **구분 필드**: `sourceType: 'MONTHLY'` (기본값)

## 📋 필드 설계안

### `transactionData` 항목 필드 확장

```javascript
{
  // 기존 필드
  id: timestamp,
  date: 'YYYY-MM-DD',
  user: '담당자명',
  type: 'income' | 'expense',
  item: '항목명',
  category: '카테고리',
  amount: number,
  paymentMethod: 'credit' | 'debit' | 'transfer' | 'cash',
  paymentDetail: '결제수단 상세명',
  merchant: '사용처',
  detail: '세부내용',
  status: '상태',
  timestamp: number,
  
  // 신규 필드 (데이터 출처 구분)
  sourceType: 'MONTHLY' | 'AUTO_CARD' | 'ACCOUNT',  // 데이터 출처
  // 'MONTHLY': 월별 현황 수동 입력 (기본값)
  // 'AUTO_CARD': 월별 현황 자동 입력 (카드 명세서 파싱)
  // 'ACCOUNT': 계좌 관리에서 등록 (사용 안 함, 별도 배열로 분리)
  
  // 레거시 호환 필드 (점진적 제거 예정)
  isCardPayment: boolean,  // 계좌 관리 카드 대금 표시 (deprecated)
  isAutoGenerated: boolean // 자동 생성 여부 (deprecated)
}
```

### 계좌 관리 카드 대금 전용 구조

**옵션 A: 별도 배열 `accountCardPayments`**

```javascript
// localStorage: 'accountCardPayments'
[
  {
    id: timestamp,
    date: 'YYYY-MM-DD',  // 결제일
    cardName: '카드명',
    cardCompany: '카드사명',
    paymentAccount: '결제 계좌명',
    amount: number,  // 청구 총액
    billingPeriod: {
      start: 'YYYY-MM-DD',
      end: 'YYYY-MM-DD'
    },
    timestamp: number
  }
]
```

**옵션 B: `accountData` 내부 필드**

```javascript
// accountData[x].cardPayments 배열
{
  id: timestamp,
  name: '계좌명',
  type: 'bank',
  balance: number,
  cardPayments: [  // 카드 대금 청구 내역
    {
      id: timestamp,
      date: 'YYYY-MM-DD',
      cardName: '카드명',
      amount: number,
      // ...
    }
  ]
}
```

**권장**: 옵션 A (별도 배열) - 계좌별로 분리되어 있어 조회/관리 용이

## 🔄 데이터 흐름 설계

### 월별 현황 수동 입력
```
entry-form submit
  ↓
newEntry 생성
  sourceType: 'MONTHLY' (기본값)
  ↓
transactionData.push(newEntry)
  ↓
saveData()
  ↓
if (paymentMethod === 'transfer' || 'debit')
  calculateAccountBalances()  // 계좌 잔액 반영
```

### 월별 현황 자동 입력
```
handleExcelImport()
  ↓
importToTransactions(transactions)
  ↓
newEntry 생성
  sourceType: 'AUTO_CARD'
  ↓
transactionData.push(newEntry)
  ↓
saveData()
  ↓
// calculateAccountBalances() 호출 안 함
```

### 계좌 관리 카드 명세서 등록
```
account-transaction-card-form submit
  ↓
newEntry 생성
  sourceType: 'ACCOUNT' (또는 별도 구조)
  ↓
accountCardPayments.push(newEntry)  ← transactionData 아님
  ↓
saveAccountCardPayments()
  ↓
calculateAccountBalances()  // 계좌 잔액 반영
```

## 🎨 필터링 규칙 설계

### 월별 현황 렌더링 필터

```javascript
function filterMonthlyTransactions(data) {
  return data.filter(entry => {
    // 계좌 관리에서 온 거래 제외
    if (entry.sourceType === 'ACCOUNT') return false;
    if (entry.isCardPayment === true) return false;  // 레거시 호환
    if (entry.category === '카드대금' && entry.paymentMethod === 'transfer') return false;  // 레거시 호환
    
    // 월별 현황에서 온 거래만 포함
    return entry.sourceType === 'MONTHLY' || 
           entry.sourceType === 'AUTO_CARD' ||
           !entry.sourceType;  // 레거시 데이터 (기본값으로 간주)
  });
}
```

### 계좌 관리 렌더링 필터

```javascript
function filterAccountTransactions(data) {
  return data.filter(entry => {
    // 계좌 관리 카드 대금 포함
    if (entry.sourceType === 'ACCOUNT') return true;
    if (entry.isCardPayment === true) return true;  // 레거시 호환
    
    // 월별 현황에서 온 거래 중 transfer/debit만 포함
    if (entry.sourceType === 'MONTHLY' || !entry.sourceType) {
      return entry.paymentMethod === 'transfer' || entry.paymentMethod === 'cash';
    }
    
    // 자동 입력 거래는 제외
    if (entry.sourceType === 'AUTO_CARD') return false;
    
    return false;
  });
}
```

## 📝 마이그레이션 계획

### 단계 1: 필드 추가 (하위 호환)
- 기존 데이터에 `sourceType` 필드가 없으면 기본값 `'MONTHLY'`로 간주
- `isCardPayment: true` → `sourceType: 'ACCOUNT'`로 마이그레이션

### 단계 2: 계좌 관리 카드 대금 분리
- `accountCardPayments` 배열 생성
- 기존 `transactionData`에서 `isCardPayment: true` 항목을 `accountCardPayments`로 이동

### 단계 3: 필터링 로직 통합
- 공통 필터 함수 생성
- 모든 렌더 함수에서 공통 필터 사용

### 단계 4: 레거시 필드 제거
- `isCardPayment`, `isAutoGenerated` 필드 제거 (선택적)
